# Team Loop 使用手册

## 1. 账号与视图

系统的访问方式由管理员统一配置：

| 身份 | 默认能力 |
| --- | --- |
| 访客 | 无需登录，具体可查看模块由管理员配置，始终为只读 |
| 类型用户 | 管理员可预设任意用户类型，并分别设置模块的查看、新增、修改和删除权限 |
| 管理员 | 管理账号、用户类型、访客范围、系统配置、备份和审计 |

管理员可以在左下角切换“管理视图/用户视图”。用户视图用于检查日常使用界面，不会改变管理员账号本身的权限。

登录用户可通过左下角“修改密码”更新自己的密码，并查看或撤销其他登录设备。账号通常建议使用员工工号，姓名显示在成员卡片中。连续登录失败达到系统上限后会临时锁定，应等待锁定结束或联系管理员排查。

管理员可在“系统管理 → 系统配置 → 企业 SSO 登录”接入 OAuth2/OIDC 企业身份平台。支持 OIDC Issuer 自动发现，也支持手动填写授权、Token、UserInfo 三个端点。配置工号字段后，系统会优先用 SSO 返回的工号关联“用户管理”中的现有账号；未匹配时可按配置自动建号并分配默认用户类型。

启用“首页自动 SSO 登录”后，未登录用户打开首页或会话自然失效时会直接跳转企业登录。若企业登录失败，页面会自动回退到系统账号登录且不会循环跳转；用户也可主动退出后使用系统账号。管理员应至少保留一个可用的系统管理员账号，作为身份平台故障时的应急入口。

管理员在“用户管理”中先创建类型，再点击类型卡片上的“配置”进入弹窗。配置分为两部分：

- 模块权限：决定可查看、新增、编辑或删除哪些模块数据；
- 业务参与名单：分别决定是否展示在团队成员、是否进入早例会跟踪、红黑榜名单和 Thank You 名单。

两部分互相独立。例如合作方可以查看红黑榜，但不必成为积分对象。保存前可评估受影响账号并预览该类型的实际视图。账号表支持按账号、工号或姓名搜索，并按团队、用户类型和授权方式筛选；勾选账号后可在统一批量工具栏中调整类型、迁移团队或批量删除，表头全选只作用于当前筛选结果。当前登录账号不会进入批量选择。单个账号资料通过“编辑”弹窗维护，批量删除的账号可由管理员在回收站恢复。删除类型前必须先迁移该类型下的有效用户；访客模板不可删除，但可随时调整只读范围。

### 大团队与组织层级

管理员在“用户管理 → 团队层级”维护组织树。系统默认提供 `ESS → MO/ES/SN`，以及 `MO → WS/RS/WH/RH` 作为示例，可按实际部门继续新增、改名和排序。每个层级包含：

- 路由标识：组成独立访问路径，例如 `/org/ess/mo/ws`；
- 可见范围：可看全部组织、当前层及下级，或仅当前层；
- 默认用户类型：SSO 自动创建账号时使用；
- SSO 群组：填写身份平台返回的群组名称，多个值用逗号分隔。

组织归属与用户类型相互独立：组织决定“能看到哪些人的数据”，用户类型决定“能看哪些模块、能执行哪些操作、是否进入业务名单”。管理员可勾选多个账号后批量调整组织。删除组织前必须先迁移其有效用户并删除下级组织。

侧栏的团队选择器只显示当前账号允许访问的层级。选择后地址栏会进入对应组织路由，成员、论坛、早例会、会议、排班、红黑榜和 Thank You 都会按该范围重新加载。上级设置为“当前层及下级”时可查看下级团队汇总；下级设置为“仅当前层”时只查看本团队。

组织透传遵循“可见与可操作分离”：

- 上级组织创建的会议会向全部下级团队透传，下级显示“上级安排”并只读；需要修改时必须切换到会议所属组织；
- 上级发布的“团队公告”会向下级透传，下级成员可以回复和使用表情，但不能编辑、删除或置顶原公告；普通讨论不会跨团队下发；
- 早例会、排班、红黑榜和成员画像仍按人员所属团队统计，不因上级可见而改变数据归属；
- 规则和常用链接是全组织共享资料；
- 同一根组织下可以跨团队 Thank You。感谢动态会在发送方团队、接收方团队以及双方共同上级显示，无关兄弟团队不可见；Thank You 之星只计入接收方团队及其上级汇总，避免在发送方团队重复排名。

SSO 配置中的“群组字段”默认为 `groups`，也支持点分字段路径。登录时系统使用最深层匹配结果确定团队并重定向到对应路由；若已有用户本次没有匹配到群组，会保留管理员手工设置的组织，避免身份平台临时漏发群组时误迁移账号。

## 2. 团队成员与团队讨论区

成员卡片展示姓名、账号、岗位、标签、职责、技能和擅长问题。点击“编辑资料”可以在弹窗中更新允许编辑的字段并上传本地头像。

团队讨论区支持：

- 按“经验分享、问题求助、项目动态、吐槽广场、公告”等分类发布主题；
- 按关键词、分类、最新更新或热度检索讨论；
- 打开主题详情后回复主题或楼中回复；
- 对主题和任意层级回复使用快捷表情；
- 点击“表情”可从完整 Emoji 分类库中搜索和选择，表情数据随系统本地部署，不依赖外网；
- 作者可以编辑或删除自己的主题、删除自己的回复，管理员可以置顶、标记已解决和执行内容治理；
- 删除的主题进入回收站，管理员可以恢复或彻底清除；
- 长内容自动换行，主题列表和详情内容均支持独立滚动。

不要在团队讨论区发布密码、客户隐私或其他敏感信息。公告分类仅管理员可用。

## 3. 工作台

工作台只展示与当前用户相关的数据：个人红黑榜积分、收到的 Thank You、个人工时和早例会事项。

工作台与红黑榜顶部的公共时间筛选首次进入时默认选择“本月 1 日至今天”，仍可按需要手动调整开始和结束日期。

早例会事项可以直接在工作台新增或更新，数据会与“早例会”模块同步。进行中和已完成事项都使用独立滚动区域；本月日历用彩色横线表示事项持续时间，点击事项后会高亮对应横线并展示每日进展。

## 4. 早例会

### 新增事项

填写事项点、今日进展/下一步、风险说明、到期时间，并直接点击状态和优先级按钮。建议一条事项只描述一个可追踪目标。

### 跨日继承

- 未完成事项会自动继承到下一天；
- 已结束日期只读，不能补改历史状态；
- 系统显示事项从哪天开始、持续多少天；
- 点击事项可在弹窗中查看有实际更新的每日进展。

### 状态建议

- `待处理`：尚未开始；
- `进行中`：已有明确推进动作；
- `有风险`：存在阻塞、依赖或时间风险；
- `已完成`：目标已交付或关闭。

## 5. 会议沙盘

会议按照“草稿、待召开、进行中、已结束、已归档”流转。管理员点击阶段条切换状态；已结束和已归档会议默认锁定，重新开启后才能修改。

### 会前

1. 拥有“会议沙盘-新增”权限的登录用户都可以创建会议，并填写日期、开始时间、会议名称和说明；管理员还可以按规则批量生成会议。
2. 创建成功后在弹窗中勾选二级预设议题。议题按一级分类分组，可搜索、整组勾选，并为每条议题指定责任人；预设较多时仅列表区域滚动，标题、搜索和提交按钮始终可见。
3. 只有管理员可以维护一级议题分类和二级预设议题；普通成员只能从已有预设中选择，或提交自定义议题。
4. 为议题明确负责人、预计时长、期望产出和会前材料。
5. 可使用“沿用上场议题”，重复标题会自动跳过。

### 会中

- 按单列议程从上到下讨论；
- 管理员可拖动议题调整顺序；
- 使用“记结论、记风险、待确认”快捷进入纪要弹窗；
- Thank You 议题自动读取本周系统数据，无需重复填写；
- 管理员在“参会签到”弹窗中登记出席、请假、迟到或缺席，并记录乐捐金额与收款状态；主界面只保留签到摘要。

### 会后

- 未讨论完的议题可顺延到下一场会议；
- 生成会议邮件前可选择是否附带本周 Thank You 内容，系统随后复制排版好的 HTML 纪要并尝试唤起默认邮件客户端；
- 将会议设为“已结束”后锁定内容，确认无误后可归档。

## 6. 机台排班

有排班查看权限的成员在月历中查看排班，自己的班次会高亮。悬停或点击日期可查看机台和支撑人员详情。

管理员可以：

- 新增或删除机台；
- 选择日期范围批量排白班/夜班；
- 删除单条排班；
- 按月份查看成员工时统计。

批量添加排班成功后月历会刷新，但已选择的结束日期会保留，便于连续安排相同时间范围；重新点击其他日期时，开始和结束日期才会同步到新日期。

删除机台会同时删除该机台下的排班，操作前应确认。

系统会阻止同一成员同日的重复班次，以及超过“单人单日最大工时”的整批排班；出现冲突时不会只保存其中一部分。

## 7. 红黑榜

红榜和黑榜细则分栏显示并带序号。管理员记录积分时先选择对应细则，再填写成员、积分和事实依据。

年度看板按成员展示 12 个月的红榜分与黑榜分，两者独立统计，不计算净分。积分明细允许管理员编辑当天记录。

进入红黑榜时，页面时间筛选默认选择当月。积分明细按时间倒序展示筛选范围内的全部记录，并在固定高度区域内滚动；点击年度看板中的成员姓名，可在弹窗中查看该成员全部历史积分和事实依据。

管理员可在年度看板右上角分别设置“公开黑榜积分”和“公开黑榜明细”。关闭后，普通用户及访客的接口和页面都不会返回对应黑榜数据；管理员管理视图仍可查看完整数据并随时恢复公开。

## 8. Thank You

每位成员每周最多感谢系统配置允许的人数，支持在下拉框中复选多人，并为每位被感谢人使用相同的事实依据。

当天送出的感谢可以修改或删除。Thank You 之星通过年份和月份下拉框查看，默认展示本月，前三名显示红心标识。

## 9. 常用链接

具备常用链接相应操作权限的登录用户可以新增、修改和删除链接。链接列表默认按以下顺序展示：

1. 置顶链接；
2. 其他链接按访问量降序；
3. 失效链接置后。

支持按分类、状态和关键词搜索。适用范围标签会根据标题、地址和说明自动建议，用户可手动修正。行尾小圆点菜单提供修改和删除操作，删除前会弹窗确认。

## 10. 搜索归档

“搜索归档”可以跨年份检索会议、议题与纪要、团队讨论主题及回复、早例会事项。搜索结果会自动遵守当前用户的模块权限，不会显示无权访问的内容。

## 11. 系统管理

管理员可以维护：

- 系统名称、团队名称和业务默认值；
- 用户类型、每个模块的操作权限和四类独立业务参与名单；
- 审计日志与回收站；
- 每日自动备份、手动备份、备份校验和指定备份恢复。

恢复数据库会覆盖当前正式数据。执行前应先下载或创建当前数据库备份，并确认没有其他用户正在写入。
